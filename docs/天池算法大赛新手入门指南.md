# 天池算法大赛 - 阿里云赛制新手入门指南

## 1. 📌 赛题概览
这是一个 **Research Agent 智能体开发比赛**，要求在阿里云 PAI-LangStudio 平台上，用通义千问（Qwen）模型构建一个能够自主搜索、推理、整合信息并回答问题的智能体。

---

## 2. 🚀 开始前的准备

### 2.1. 1️⃣ 注册与资源领取
- 前往 **[阿里云免费试用中心](https://free.aliyun.com/?productCode=learn)**
- 领取以下免费资源：
  - **PAI-DSW**（交互式建模）
  - **PAI-EAS**（模型服务部署）
- 开发初期推荐使用 **2C4G** 机型，后续可根据需要升级

### 2.2. 2️⃣ 了解比赛流程
比赛分两个阶段：

| 阶段 | 时间 | 内容 | 目标 |
|------|------|------|------|
| **阶段一** | ~3月9日 | 用100道公开题目调试验证 | 排名前60%进入阶段二 |
| **阶段二** | 3月9日-13日 | 用100道隐藏题目正式评测 | 最终排名 |

---

## 3. 💡 比赛任务详解

### 3.1. 你的 Agent 需要做到：
1. **接收问题**：通过 HTTP POST 接收自然语言问题
2. **自主搜索**：调用搜索引擎查找答案
3. **推理整合**：分析搜索结果，提炼答案
4. **返回结果**：以 JSON 格式返回答案

### 3.2. API 接口规范：
```json
// 输入
{
  "question": "Where is the capital of France?"
}

// 输出
{
  "answer": "巴黎"
}
```

---

## 4. 🛠️ 技术栈与限制

### 4.1. ✅ 允许使用的资源：
- **模型**：通义千问（Qwen）系列
  - PAI Model Gallery 部署的模型
  - 阿里云百炼提供的模型服务
- **工具**：搜索引擎（Bing、SerpAPI 等）、MCP 协议工具

### 4.2. ❌ 严格禁止：
- ❌ **模型微调**（包括 LoRA、Adapter 等）
- ❌ **硬编码答案**（不能提前知道题目答案）
- ❌ **调用第三方 Agent 服务**（所有逻辑必须在 LangStudio 中实现）
- ❌ **作弊行为**（如替换模型、哈希映射等）

---

## 5. 📋 交付物清单

### 5.1. 阶段一提交：

## 6. 🎯 新手快速上手步骤

### 6.1. Step 1：熟悉平台
- 学习 [PAI-LangStudio 快速入门](https://help.aliyun.com/zh/pai/user-guide/langstudio/)
- 了解工作流模式和代码模式的区别

### 6.2. Step 2：下载验证集
- 下载 [100道验证题](https://pai-quickstart-predeploy-hangzhou.oss-cn-hangzhou.aliyuncs.com/tianchi/question.jsonl)
- 本地调试你的 Agent

### 6.3. Step 3：设计 Agent 架构
推荐参考经典 Agent 模式：
- **ReAct 模式**：推理-行动循环
- **Plan-and-Execute**：先规划再执行

### 6.4. Step 4：开发与测试
- 在 LangStudio 中搭建工作流
- 用验证集测试正确率
- 优化提示词和工具调用策略

### 6.5. Step 5：部署服务
- 将应用流部署为 EAS 服务
- 确保接口符合规范（30分钟超时内返回）

### 6.6. Step 6：提交与等待结果
- 阶段一提交答案文件
- 进入前60%后提交服务信息
- 等待阶段二评测结果

---

## 7. ⚠️ 重要提醒

1. **作弊检测严格**：主办方会检查 EAS 服务是否与提交的项目包一致
2. **只有一次机会**：阶段二评测只有一次测试机会，服务异常=零分
3. **答案格式重要**：必须严格遵循题目要求的格式
4. **注意时间节点**：3月9日是阶段一和阶段二提交的截止日期
